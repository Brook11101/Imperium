# Imperium 技术文档索引

- 阶段：技术设计
- 范围：从产品方案到工程落地的中间层文档
- 语言：中文

## 使用说明

本目录承载 `Imperium` 的正式技术方案文档。

拆分原则如下：

- 每个文档聚焦一个技术层面
- 当前阶段只定义“系统怎么建”，不直接写实现代码
- 所有技术方案默认服务于 MVP 可落地，而不是理论最优

## 文档列表

### `01-总体架构与技术选型.md`

回答问题：

- 系统整体怎么拆
- 为什么选这些技术
- 运行单元怎么划分

### `02-领域模型与状态机实现.md`

回答问题：

- 领域对象怎么设计
- 状态机怎么落地
- 运行模式怎么映射到业务规则

### `03-数据存储与消息事件设计.md`

回答问题：

- 数据落在哪些表里
- 事件怎么发
- 如何保证一致性、顺序性、幂等性

### `04-Agent集成与调度执行设计.md`

回答问题：

- Java 服务如何调用 OpenClaw CLI
- Agent 调度如何异步执行
- 停滞、重试、升级、回滚如何设计

### `05-前后端接口与页面技术方案.md`

回答问题：

- 前端如何搭建
- API 怎么分层
- 页面如何映射到业务能力

### `06-部署方案与非功能设计.md`

回答问题：

- Docker 怎么部署
- 可靠性、可观测性、安全性如何保障
- 当前研发阶段的非功能底线是什么

## 建议阅读顺序

1. 先读 `01-总体架构与技术选型.md`
2. 再读 `02-领域模型与状态机实现.md`
3. 接着读 `03-数据存储与消息事件设计.md`
4. 然后读 `04-Agent集成与调度执行设计.md`
5. 再看 `05-前后端接口与页面技术方案.md`
6. 最后看 `06-部署方案与非功能设计.md`

## 当前阶段边界

当前阶段不做：

- 具体 Controller / Service / Mapper 编码
- 完整 SQL 执行脚本
- OpenClaw Agent 的 Prompt 定稿
- 页面组件级交互稿
- 监控平台完整配置

当前阶段产出将作为后续编码阶段的直接输入。
