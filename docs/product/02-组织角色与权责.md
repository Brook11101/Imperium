# 组织角色与权责

## 1. 设计原则

角色体系的目标不是做世界观装饰，而是把不同类型的权力拆开。

在 `Imperium` 中，至少要清晰拆开这几类权力：

- 接入权
- 议政权
- 否决权
- 裁决权
- 行政执行权
- 审计权
- 归档权

这样做的原因是：

- 让系统内部形成真实制衡
- 防止所有 Agent 都变成“万能助手”
- 让每种输出都能对应到明确制度角色

## 2. 角色总览

### 2.1 `Caesar`

- 身份：恺撒，即用户本人
- 定位：最终裁决者，不作为普通 Agent 实现
- 主要职责：
  - 下达法令
  - 决定运行模式
  - 批准、驳回、限制或 override 议案
  - 暂停、撤销、重开重要事务

### 2.2 `praeco`

- 身份：传令官
- 定位：入口总管
- 主要职责：
  - 接收法令
  - 判断是闲聊、查询、正式议案还是紧急军令
  - 规范化标题与摘要
  - 推荐运行模式
  - 将复杂议案送入元老院，将简单事务送往执行链
- 明确边界：
  - 不做政策判断
  - 不直接执行工作
  - 不覆盖保民官决定

### 2.3 `senator_strategos`

- 身份：战略派元老
- 定位：价值与优先级判断者
- 主要职责：
  - 评估事务是否值得推进
  - 识别机会成本
  - 给出阶段化推进建议
  - 从全局目标角度质疑或支持议案

### 2.4 `senator_juris`

- 身份：法理派元老
- 定位：约束与制度边界守护者
- 主要职责：
  - 检查议案是否越权
  - 检查是否违反既有约束
  - 识别制度性风险
  - 指出规则冲突与边界问题

### 2.5 `senator_fiscus`

- 身份：财政派元老
- 定位：成本与资源视角的审议者
- 主要职责：
  - 评估时间、资源、成本投入
  - 提供轻量版、标准版、重型版方案
  - 指出资源浪费和范围膨胀

### 2.6 `tribune`

- 身份：保民官
- 定位：前置否决权角色
- 主要职责：
  - 审查议案是否损害用户利益
  - 审查是否存在高风险、不合理边界或重大隐患
  - 对议案做通过、否决或退回重议
  - 在必要时强制进入恺撒裁决
- 明确边界：
  - 不直接执行事务
  - 不变成执行总调度者

### 2.7 `consul`

- 身份：执政官
- 定位：行政调度中心
- 主要职责：
  - 把批准议案转成执行命令
  - 分派给合适的执行角色
  - 管理依赖与进度
  - 汇总执行成果
- 明确边界：
  - 不参与元老院辩论
  - 不替代恺撒做最终裁决

### 2.8 `legatus`

- 身份：军团使节 / 工程执行者
- 定位：技术落地主力
- 主要职责：
  - 编码实现
  - 技术方案落地
  - 工程交付

### 2.9 `praetor`

- 身份：法务官 / 审判官
- 定位：质量、规则、验收角色
- 主要职责：
  - 测试
  - 验收
  - 安全与质量检查
  - 规则符合性校验

### 2.10 `aedile`

- 身份：营造官 / 市政官
- 定位：对外呈现与交付包装角色
- 主要职责：
  - UI / UX
  - 文档结构
  - 文案呈现
  - 面向用户的可交付表达

### 2.11 `quaestor`

- 身份：财务官
- 定位：成本与资源分析角色
- 主要职责：
  - 成本核算
  - 数据支撑
  - 预算与资源消耗分析

### 2.12 `scriba`

- 身份：书记官
- 定位：总结与归档角色
- 主要职责：
  - 汇总呈送恺撒的摘要
  - 将零散活动整理成正式案卷
  - 生成最终归档文书
- 明确边界：
  - 不成为调度中心
  - 不抢夺执行权

### 2.13 `censor`

- 身份：监察官
- 定位：制度监督与长期治理角色
- 主要职责：
  - 评估流程是否符合制度
  - 记录 override 与异常模式
  - 输出长期治理优化建议

### 2.14 `governor_*`

- 身份：行省总督 / 领域执行代理
- 定位：边界明确的专项执行角色
- 主要职责：
  - 在特定领域或外部系统中执行工作
  - 向执政官和书记官回报结果

## 3. 角色分层

### 3.1 主权层

- `Caesar`

### 3.2 接旨层

- `praeco`

### 3.3 议政层

- `senator_strategos`
- `senator_juris`
- `senator_fiscus`

### 3.4 否决层

- `tribune`

### 3.5 行政调度层

- `consul`

### 3.6 执行层

- `legatus`
- `praetor`
- `aedile`
- `quaestor`
- `governor_*`

### 3.7 归档与监察层

- `scriba`
- `censor`

## 4. 角色设计结论

这套角色体系的核心不是“角色很多”，而是“权力类型被拆开了”。

可以浓缩成一句话：

- 传令官负责受理，元老负责辩论，保民官负责阻断，恺撒负责裁决，执政官负责调度，专职机构负责执行，书记官负责归档，监察官负责审视制度本身。
