# 运行模式与状态机

## 1. 设计目标

状态机不是简单地表示“任务从开始做到结束”，而是表示一份事务如何在帝国制度中被处理。

因此，`Imperium` 的状态机必须满足：

- 区分下令与执行之间的政治过程
- 支持元老院辩论
- 支持保民官否决
- 支持恺撒裁决
- 支持执行完成后的审计回流

## 2. 运行模式

### 2.1 `StandardSenate`

- 默认模式
- 适用于复杂、需要权衡的任务
- 走完整制度流程

典型路径：

`EdictIssued -> Triaged -> InSenate -> Debating -> VetoReview -> AwaitingCaesar -> Mandated -> Delegated -> InExecution -> UnderAudit -> Archived`

### 2.2 `DirectDecree`

- 快速执行模式
- 适用于低风险、需求明确或紧急事务
- 跳过元老院辩论

典型路径：

`EdictIssued -> Triaged -> AwaitingCaesar -> Mandated -> Delegated -> InExecution -> UnderAudit -> Archived`

### 2.3 `TribuneLock`

- 高风险模式
- 适用于安全、生产、预算或不可逆事务
- 强化保民官审查与回流能力

典型路径：

`EdictIssued -> Triaged -> InSenate -> Debating -> VetoReview -> AwaitingCaesar -> Mandated -> Delegated -> InExecution -> UnderAudit -> AwaitingCaesar -> Archived`

### 2.4 `CampaignMode`

- 长周期战役模式
- 面向由多个阶段性议案组成的大型事务
- 当前仅保留概念，不作为 MVP 范围

## 3. 状态总览

### 3.1 核心状态

- `EdictIssued`
- `Triaged`
- `InSenate`
- `Debating`
- `VetoReview`
- `AwaitingCaesar`
- `Mandated`
- `Delegated`
- `InExecution`
- `UnderAudit`
- `Archived`
- `Rejected`

### 3.2 控制状态

- `Suspended`
- `Revoked`

## 4. 状态定义

### `EdictIssued`

- 恺撒已经发出法令
- 议案刚刚建立，尚未进入制度流程

### `Triaged`

- 传令官已完成规范化处理
- 已生成标题、摘要、模式建议和基础分类

### `InSenate`

- 议案进入元老院议程
- 尚未开始正式辩论

### `Debating`

- 元老院正式辩论中
- 多个元老从不同立场提交意见

### `VetoReview`

- 保民官对议案做制度性审查

### `AwaitingCaesar`

- 议案已达到必须由恺撒拍板的节点

### `Mandated`

- 议案已被正式授权
- 具备进入执行链的政治合法性

### `Delegated`

- 执政官已完成分派
- 形成清晰的执行命令

### `InExecution`

- 执行角色正在完成工作

### `UnderAudit`

- 执行结果正在接受审计、验收、风险复核或最终文书整理

### `Archived`

- 议案已完成并归入档案

### `Rejected`

- 议案被制度性否决

### `Suspended`

- 议案被暂停，允许后续恢复

### `Revoked`

- 恺撒主动撤销议案

## 5. 合法流转

### 5.1 主流转

- `EdictIssued -> Triaged`
- `Triaged -> InSenate`
- `Triaged -> AwaitingCaesar`
- `Triaged -> Mandated`
- `InSenate -> Debating`
- `Debating -> VetoReview`
- `VetoReview -> AwaitingCaesar`
- `VetoReview -> InSenate`
- `VetoReview -> Rejected`
- `AwaitingCaesar -> Mandated`
- `AwaitingCaesar -> InSenate`
- `AwaitingCaesar -> Revoked`
- `Mandated -> Delegated`
- `Delegated -> InExecution`
- `InExecution -> UnderAudit`
- `UnderAudit -> Archived`
- `UnderAudit -> InExecution`
- `UnderAudit -> AwaitingCaesar`

### 5.2 控制流转

- 任意非终态可转入 `Suspended`
- `Suspended` 恢复到之前状态
- 任意非终态可由恺撒转入 `Revoked`

## 6. 状态责任角色

建议责任映射如下：

- `EdictIssued`：`Caesar`
- `Triaged`：`praeco`
- `InSenate`：系统 / 元老院机制
- `Debating`：各元老
- `VetoReview`：`tribune`
- `AwaitingCaesar`：`Caesar`
- `Mandated`：`consul`
- `Delegated`：`consul`
- `InExecution`：执行角色
- `UnderAudit`：`praetor` + `scriba`
- `Archived`：`scriba`
- `Rejected`：`tribune` 或 `Caesar`

## 7. 设计结论

`Imperium` 的状态机不是执行流水线，而是“议案制度生命周期”。

一句话总结：

- 每项事务先成为法令，再成为议案；先被辩论，再被裁决；先被授权，再被执行；最后被审计并写入帝国档案。
